# 翻译性能优化说明

## 问题诊断

### 当前情况
```
1580条字幕，batch_size=5
总批次: 316批
每批耗时: 20-28秒
预计总时间: 约2小时
```

### 性能瓶颈分析

#### 1. **批量大小太小** ⚠️
- 之前：5条/批 → 316批
- 问题：API调用次数过多

#### 2. **Prompt太冗长** ⚠️
- 之前的prompt：
```
你是一个专业的字幕翻译专家。

翻译风格要求：
...长篇大论...

请将以下英文字幕翻译为中文，只输出翻译结果，不要添加任何解释或标注：

{text}

中文翻译：
```
- 问题：每次都要处理大量上下文tokens

#### 3. **无输出限制** ⚠️
- 之前：没有`num_predict`限制
- 问题：Ollama可能生成超长输出，浪费时间

#### 4. **Ollama本身速度** ⚠️
- qwen3:8b模型：约1500-2000 tokens/分钟
- 这是正常的本地LLM速度

## 优化方案

### ✅ 优化1：增加批量大小

```python
# 之前
batch_size = 5  # 316批，约2小时

# 优化后
batch_size = 10  # 158批，约1小时
```

**效果预估**：
- 批次数量减半：316 → 158批
- 时间减半：约1小时

### ✅ 优化2：简化Prompt

```python
# 之前（冗长）
prompt = """你是一个专业的字幕翻译专家。

翻译风格要求：
{style}

请将以下英文字幕翻译为中文，只输出翻译结果，不要添加任何解释或标注：

{text}

中文翻译："""

# 优化后（精简）
prompt = """字幕翻译（英译中）
风格：{style}

{text}

翻译："""
```

**效果**：
- Tokens减少：约50-70%
- 处理时间减少：约15-20%

### ✅ 优化3：限制输出长度

```python
# 添加num_predict限制
estimated_tokens = len(text) * 2  # 中文比英文短
max_tokens = min(estimated_tokens, 2000)

options={
    'num_predict': max_tokens,  # 限制输出
    'top_p': 0.9,  # 提高速度
    'top_k': 40,
}
```

**效果**：
- 防止过度生成
- 加快每批完成速度：约10-15%

### ✅ 优化4：改进日志和进度显示

```python
logger.info(f"批次大小: {batch_size}条/批，总批次: {total_batches}")
logger.info(f"估计时间: {total_batches * 10 / 60:.1f}-{total_batches * 18 / 60:.1f} 分钟")
```

### 可选优化3：并发翻译（高级）

**警告**：需要修改代码，较复杂

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

# 使用线程池并发处理多个批次
with ThreadPoolExecutor(max_workers=2) as executor:
    futures = []
    for batch in batches:
        future = executor.submit(translate_batch, batch)
        futures.append(future)
```

**效果**：
- 可能再快50%
- 需要更多RAM和CPU

**风险**：
- 可能超出Ollama资源限制
- 需要仔细的同步控制
